NEW ARCHITECTURE I WANT

PART 1 — STRUCTURED SITE GENERATION PIPELINE

Build the website generator as a pipeline with these stages:

Stage A: Interview / intake
The AI interview should collect enough structured information to build a real site.
Gather:
- organization type
- mission
- audience
- programs/services
- recurring events
- membership/join info
- contact details
- meeting location and schedule
- tone/personality
- colors/branding
- logo / photo availability
- key calls to action
- optional history, board, sponsors, partners, donation info

Stage B: Site spec
Convert the interview into a structured site spec JSON.
This should include:
- organization profile
- recommended pages
- navigation structure
- page goals
- section recommendations per page
- CTA strategy
- content blocks
- event and sponsor display rules
- brand settings

Stage C: Page planner
Before rendering anything, generate a page plan.
Do not assume every organization gets the same pages.
Choose pages based on organization type.

Examples:
Rotary / civic org:
- Home
- About
- Service Projects or Programs
- Events
- Membership / Join
- Contact

Festival / event org:
- Home
- Event Info
- Vendors
- Sponsors
- FAQ
- Contact

General nonprofit:
- Home
- Mission
- Programs
- Events
- Get Involved
- Contact

Stage D: Module composition
Each page should be built from reusable modules, not one hard-coded layout.
Create a modular design system with components such as:
- hero variants
- intro sections
- card grids
- event lists
- timeline blocks
- stats sections
- CTA bands
- sponsor strips
- testimonials
- FAQ sections
- board/member highlights
- gallery/media sections
- contact panels
- join/donate/volunteer sections

The AI should choose which modules to use and in what order based on the site spec.

PART 2 — DESIGN SYSTEM / CREATIVE VARIATION

I want visual variation without sacrificing quality.

Create a strong website design system with:
- multiple high-quality hero variants
- multiple page layout patterns
- multiple section compositions
- strong typography hierarchy
- better spacing rhythm
- calmer, more premium SaaS-quality aesthetics
- proper image treatment
- cleaner navigation/header behavior
- mobile-first responsiveness

Do not let every site look identical.
But also do not let the output become chaotic.

The right approach is:
AI selects from a curated system of high-quality compositions.

Examples of variation that should be possible:
- left-text / right-image hero
- centered hero with background image
- split hero with stats
- mission-first nonprofit homepage
- event-first civic homepage
- membership-first service club homepage

PART 3 — CONTENT QUALITY

The AI should not generate shallow filler copy.
Improve content generation so that:
- it writes more specific copy
- it reflects the organization type
- it avoids generic nonprofit buzzwords
- it does not invent fake stats unless clearly marked placeholders
- it surfaces missing info instead of making things up

If the org did not provide real stats, do not fake “200+ members” or “1985 founded.”
Either omit those sections or ask for the data.

PART 4 — IMAGES / MEDIA

Stop using a tiny fixed list of stock photo IDs.

New rules:
- prefer organization-uploaded images and logos whenever available
- support placeholders gracefully if no media exists
- if stock imagery is used, it should be broader, more relevant, and selected by theme
- image choice should match org type
- do not make every site look like the same community-stock-photo website

Also:
make the design still look strong even when the organization has no good photos.

This should work like discoverirwin.com:
the site should reflect live data from the platform, not just frozen generated content.

Stop presenting Pillar as a feature dashboard and start presenting it as an Organization Command Center
Right now the UI likely feels like:
"Here are the tools you can use."
It should feel like:
"Here is what your organization needs attention on today."
This is the difference between:
Software UI → Feature navigation
Operating system UI → Situation awareness
And you are building an operating system.
The biggest UX mistake most SaaS makes (and you are close to it)
Most dashboards look like:
Events
Contacts
Website
Payments
AI
Settings
That is product-centric navigation.
Your users don't think like that.
Rotary leaders think like:
What needs done?
What is broken?
What is coming up?
Who hasn't paid?
What is overdue?
That is operations-centric thinking.
What your dashboard should look like instead
The homepage after login should be:
Organization Health Dashboard
Top section:
Today at a glance
Upcoming events
Pending approvals
Unpaid invoices
Upcoming deadlines
AI recommendations

Organization Status

Social post awaiting review
Board vote required
Then:
Upcoming

Renew insurance
Send newsletter
Then:
AI Suggestions
Suggested sponsor outreach email
Suggested social post
Suggested volunteer request
Why this matters more than design polish
Because this changes Pillar from:
"Software you log into"
Into:
"The place you run your organization."
That is a massive psychological shift.
And that is what makes SaaS sticky.
The UX principle you should adopt
Everything should answer:
"What should I do next?"
Not:
"What can this system do?"
Features belong in side nav.
Decisions belong on the dashboard.

Key Advantage You Have Over Us
Since your forms, database, and dashboard are all one system, you eliminate every integration pain point we deal with:

No data sync delays — we cache Google Sheets data for 5 minutes. You read from the database in real time.
No URL breakage — we had to build a permanent download system because JotForm changed their auth. Your logos are local from submission.
No note parsing — we parse free-text Square transaction notes to figure out event attribution. Your Stripe metadata is structured.
No multi-system auth — we manage Google OAuth, JotForm API keys, Apps Script keys, and session auth. You just need session auth and Stripe keys.
No event sync endpoint — we need POST /api/sync-event because the website and dashboard are separate. If yours are one system, events exist in one place.

Vendor Approval Flow
Dashboard reads vendors:
GET /api/vendors?event=<slug>

Returns all vendors for that event with approval status and document status. Requires admin auth.

Admin approves:
PATCH /api/vendors/:id
{ "approved": "yes" }

Updates the row directly. Optionally send a welcome/confirmation email via SendGrid.

Admin rejects:
PATCH /api/vendors/:id
{ "approved": "no", "rejection_reason": "Missing health department permit" }

Updates the row AND triggers a SendGrid rejection email to the vendor with the reason.

Admin marks document received:
PATCH /api/vendors/:id/documents
{ "servsafe_received": true }

Sets the flag. Clears future reminders for that document.

Automated Document Reminders (background job):
Run a scheduled job every hour:

SELECT v.*, e.date as event_date
FROM vendors v
JOIN events e ON v.event_id = e.id
WHERE v.approved = 'yes'
AND e.active = true
AND (v.servsafe_received = false OR v.insurance_received = false)

For each result:

Calculate days until event
If more than 7 days out → send reminder only if last_servsafe_reminder is older than 7 days
If within 7 days → send reminder only if last_servsafe_reminder is older than 1 day
After sending, update last_servsafe_reminder or last_insurance_reminder to now
On server startup, delay reminders by 6 hours to prevent spam on restarts
Email via SendGrid:

To: vendor.email
Subject: "Reminder: Missing documents for {event.name}"
Body: "Hi {vendor.contact_name}, you are approved for {event.name} on {event.date} but we still need your {missing_doc}. Please submit it as soon as possible."

Financial Tracking (Stripe)
Stripe Webhook Listener:
POST /api/stripe-webhook

Verify the Stripe signature, then handle these events:

payment_intent.succeeded:

const metadata = event.data.object.metadata;
const eventSlug = metadata.event_slug;

// Look up event_id from slug
// Insert into payments table
// If category === "ticket", also insert into tickets table

payment_intent.payment_failed:

Insert into payments with status = 'failed'
Do NOT count in revenue
charge.refunded:

Update existing payment to status = 'refunded'
Subtract from revenue

Key Lessons From the Reference Architecture
Permanent file storage for uploaded images. Never depend on third-party URLs. Save locally on upload. Name files by MD5 hash to avoid duplicates.
Slug-based event attribution everywhere. Every vendor, sponsor, payment, ticket, and registration ties back to an event via slug. This makes multi-event support trivial.
Structured Stripe metadata over note-parsing. The reference system's biggest pain point is parsing free-text Square transaction notes. With Stripe metadata, attribution is structured from the start. Always include event_slug and category.
Public vs. authenticated endpoint boundary. Sponsor logos and ticket stats are public. Everything else requires admin auth. Keep this separation clean.
Automated reminders with backoff. Track when the last reminder was sent. 7-day interval normally, 1-day when within a week of the event. 6-hour delay on server startup.
Cache expensive queries. Even with a database instead of Google Sheets, cache aggregated revenue and sponsor data for at least 60 seconds to avoid hammering the DB on every dashboard load.
Absolute URLs in public endpoints. Any API that an external website consumes must return full absolute URLs for images and resources, not relative paths.
Deduplication by email. When aggregating contacts across vendors, sponsors, registrations, and tickets, always deduplicate by email address before displaying or emailing.

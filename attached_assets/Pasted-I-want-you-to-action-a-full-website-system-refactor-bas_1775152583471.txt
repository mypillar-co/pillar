==================================================
CORE GOAL
==================================================

Build the website system so that:

- websites are generated from structured records, not one giant HTML blob
- generated_html remains only a compiled artifact for preview/publish compatibility
- website content remains editable after generation
- site import is structured, selective, and high-quality
- dynamic modules bind to real operational data
- event behavior is explicit and public-site-aware
- auto-update is modular, conservative, and auditable
- dashboard modules do NOT connect directly to the renderer
- everything flows through a site data integration layer

This aligns with the Phase 1 overhaul direction and the current dashboard integration map already identified. :contentReference[oaicite:0]{index=0}

==================================================
NON-NEGOTIABLE PRINCIPLES
==================================================

1. Structured records are the source of truth
2. generated_html is compiled output only
3. Import must curate and map, not scrape and rewrite
4. Site planning must happen before rendering
5. AI should assist extraction, planning, and drafting — not generate arbitrary HTML/CSS
6. Event behavior must be explicit, not inferred ad hoc throughout the app
7. Auto-update must operate at the block/module level only
8. Identity content must remain stable unless explicitly changed
9. Dashboard data must never connect directly to the renderer
10. All website integrations must flow through a site data integration layer
